# Registry Candidates v1

Draft backlog for future Listing Copilot registry infrastructure.

This file is not runtime data. It summarizes recurring candidates from QA Subsets A-F and legacy training notes.

## Candidate Fields

Future registry entries should conceptually support:

- name
- family
- classification
- product scope
- commercial importance
- title inclusion default
- status

Suggested status values:

- candidate
- validated
- implemented
- deprecated
- needs ground truth
