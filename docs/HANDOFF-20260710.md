## 1. Background — why this batch of changes exists

The product converts sports-card images into eBay listing titles
(Evidence-first / Catalog-assisted / Writer-reviewed). Production:
https://listing.lyncafei.team (Vercel) + Supabase `osrrujmpxxiefppjfgpd` +
a Cloud Run recognition worker (PaddleOCR + SigLIP embeddings).

Three stacked mandates drove the last two days:

1. **GPT-5-mini regression** (2026-07-08): switching the main path from
   gpt-4.1-mini to gpt-5-mini dropped policy-fair accuracy 0.912 → 0.807.
   Root causes found and fixed: 10x out-of-spec `max_output_tokens`
   (every call ate a 400+downgrade-retry round trip), reasoning-effort
   defaults hostile to literal extraction, and a preingestion hard-evidence
   channel that had **no consumer at all** (jobs enqueued forever, evidence
   never reached titles).
2. **Speed mandate**: minimize writer-PERCEIVED latency (upload → title
   visible), authorizing framework changes and invisible preloading.
3. **全改 (fix everything)**: a whole-codebase review produced a reform
   list (security, CI, dead jobs, config debt, catalog entity resolution,
   v2-monolith retirement) — all small items shipped; the two big reforms
   (R1/R4) got battle plans and R1 is mid-execution.

## 2. What has been done

### Accuracy / model path
- gpt-5-mini fixed and adopted as production default (`OPENAI_LISTING_MODEL`):
  token caps model-legal (128k / 32,768), reasoning `minimal`, verbosity
  `medium` (A/B via `OPENAI_GPT5_REASONING_EFFORT` / `OPENAI_GPT5_TEXT_VERBOSITY`).
- Baseline (C100 first-10, take-5): **policy-fair 0.851, pass@0.72 9/10** vs
  0.807 pre-fix. Tracked in EVAL_REGISTRY.

### Perceived-latency architecture (all live)
- **Speculative recognition** (frontend `ENABLE_SPECULATIVE_RECOGNITION`):
  recognition starts at image-ready; the click attaches to the running job.
- **Exact-anchor writer fast lane** (`ENABLE_V4_EXACT_ANCHOR_WRITER_FAST_LANE`):
  the ONE L1 outcome allowed through the L1 return barrier — unique
  strictest-tier catalog hit renders the title in ~2s; L2 verifies in
  background. Verified repeatedly: **perceived 0ms** on covered cards
  (gate p50 305ms → 0ms on hits).
- **Skeleton streaming lite**: pending cards show provisional scout fields.
- Anchor-first path v1: anchor classifier + `cert_registry` cert lane in L1
  finalize (registry fed by writer-feedback flywheel; empty until real
  writer traffic; external grader lookups disabled pending licensing).

### Hard-evidence (OCR) chain — built end to end
- `preingestion-ocr-worker.mjs` + `/api/v4/listing-preingest-worker` consume
  `ocr_crop_verification` jobs; patches land on bundles and flow into title
  resolution. Verified with real reads (e.g. "BS-4 JESUS MADE" @0.98).
- Cloud Run worker fixed: PP-OCRv5 **mobile** + mkldnn OFF (server models
  SIGFPE-crashed on that CPU), models **baked into the image** (was
  re-downloading from Baidu CDN per cold start → 503s), 8Gi, min-instances=1,
  cpu-boost, 3 uvicorn processes. Worker token: GCP Secret Manager
  `lynca-recognition-worker-token` (project gen-lang-client-0465339486).
- Consumerless job types (visual_embedding/surface/quality) default OFF;
  441 dead rows cancelled; gap-queue rows have deterministic ids (551→117).

### Catalog flywheel
- **Dual-agreement promotion pump** `scripts/catalog-promote-gap-queue.mjs`:
  a gap row enters the catalog only when printed code is stable across ≥2
  recognition runs AND a human-written title agrees on player+year, with a
  recognizable product line. 20 promoted so far (7 manual + 13 pumped),
  6 rejected with explicit reasons (still queued for humans). Promoted rows
  are identity-fields-only, REVIEW_REQUIRED, fully provenanced.
- Fast-lane hits on promoted cards confirmed in production gates.

### Gate methodology (important operational knowledge)
- Local egress to Vercel edge is flaky (45s hangs that never reach a
  function). **The canonical gate is GitHub Actions `smoke-gate`**
  (workflow_dispatch; inputs limit/model/think_ms; creds in repo secrets;
  eval datasets whitelisted into the repo).
- Dispatch gates ≥7 minutes after an alias switch; a first-run all-timeout
  gate right after deploy is suspect infra → rerun once before investigating.
- **PostgREST weather** (2026-07-10 ~02:00Z window): transient REST-layer
  failures cause fast-lane misses (`exact_anchor_finalize.reason =
  catalog_lookup_unavailable` on pending L1 — diagnostic added for exactly
  this) and first-card timeouts while the DB itself is healthy. Finalize RPC
  now retries once (verified recovering a hit mid-window); gate warms the
  path before scoring. If windows recur, review Supabase tier/pooler.
